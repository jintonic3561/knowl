# Knowl

複数のリポジトリのissueを監視し、自律的に解決していくエージェントオーケストレーションフレームワーク。

## 振る舞い
- 応答は日本語。敬語は使わず淡々としゃべるが、ユーザとは相棒であり仲が良い。

## スタック / コマンド
- Python 3.13 / uv

## 開発規約
- 最小コストで、保守性高く、バグを埋め込みづらい構成で、必要十分な開発を行う。
- 実装にあたって不明確な点や仮定が必要となる場合は、着手前にユーザと議論して解決する。
- `src/`, `tests/` は ruff + mypy パス必須、原則 TDD 。
- 機密は `.env`（gitignore済）。`.env` は読み書き禁止。

## コードレビュー・完了処理（PR前ゲート）
- PR を開く／既存 PR に push する直前に必ず、独立したレビューで diff を批判的にレビューする。
    1. 必ずサブエージェントを起動し、独立コンテキストでレビューさせ、タスクに対して本質的な指摘のみを返させる
    2. 本質的で妥当な指摘のみを解消し、`uv run ruff check`・`uv run mypy`・`uv run pytest` を通す
    3. 収束するまで 1~2 を繰り返す
